## Введение

FreeCAD может импортировать и экспортировать множество форматов файлов. Для некоторых форматов существуют специальные настройки. Их можно найти в [Редакторе Настроек](/Preferences_Editor/ru "Preferences Editor/ru"). В меню выберите **Правка → Настройки...** и затем **Импорт/Экспорт**.

Не все параметры импорта и экспорта доступны по умолчанию. Для некоторых из них необходимо сначала запустить верстак.

Эта страница обновлена для версии 1.0.

## Примечания

### Страницы с техническими чертежами

Параметры DXF и SVG, перечисленные здесь, не используются командами ![](/images/TechDraw_ExportPageSVG.svg) [Экспорт листа в SVG](/TechDraw_ExportPageSVG/ru "TechDraw ExportPageSVG/ru") и ![](/images/TechDraw_ExportPageDXF.svg) [Экспорт листа в DXF](/TechDraw_ExportPageDXF/ru "TechDraw ExportPageDXF/ru") ![](/images/Workbench_TechDraw.svg) [Верстака TechDraw (Чертёж)](/TechDraw_Workbench/ru "TechDraw Workbench/ru"), или опцией экспорта в меню [Чертежа](/TechDraw_Workbench/ru "TechDraw Workbench/ru"): **Файл → Экспортировать → Технический чертёж (\*.svg \*.dxf \*.pdf)**.

### IFC файлы

Некоторые, специфические для NativeIFC, настройки импорта и экспорта IFC можно найти в разделе [Настройки BIM](/BIM_Preferences/ru "BIM Preferences/ru").

### Файлы OpenSCAD

Параметры импорта и экспорта файлов OpenSCAD можно найти в разделе [Настройки OpenSCAD](/OpenSCAD_Preferences/ru "OpenSCAD Preferences/ru").

## Доступные настройки

### DAE

![](/images/Preferences_Import-Export_Page_DAE.png)

Формат [Collada (рус.)](http://ru.wikipedia.org/wiki/COLLADA) [(англ.)](http://en.wikipedia.org/wiki/COLLADA) DAE (Digital Asset Exchange - Обмен Электронными Ресурсами) - это стандартный формат файлов для обмена данными о сетках. FreeCAD может импортировать сетки из файлов .dae, а также экспортировать объекты на основе [Форм](/Part_Workbench/ru "Part Workbench/ru") в формат .dae.

Примечание для пользователей Linux: Для работы с этим форматом файлов FreeCAD требуется модуль [pyCollada](/Extra_python_modules/ru "Extra python modules/ru").

На этой вкладке вы можете настроить следующие параметры:

| Параметр | Описание |
| --- | --- |
| **Коэффициент масштабирования** | Все размеры в файле будут масштабированы с указанным коэффициентом. |
| **Генератор полигональной сетки** | Устанавливает программу для создания сетки, которая должна использоваться. Если используется *Нетген (Netgen)*, убедитесь, что она доступна. Это можно проверить с помощью  [Верстака Полигональных сеток](/Mesh_Workbench/ru "Mesh Workbench/ru") и [создать полигональную сетку](/Mesh_FromPartShape/ru "Mesh FromPartShape/ru") с помощью Нетген. Если он недоступен, необходимо установить другую версию FreeCAD, скомпилированную с Нетген. |
| **Тесселяция (мозаика)** | Значение тесселяции для использования с программами построения сетки *Встроенный* и *Мефисто*. |
| **Дискретизация** | Значение приращения, используемое для создания сетки с помощью *Нетген*. Это значение описывает, насколько быстро уменьшается размер сетки. Градиент локального размера сетки `h(x)` ограничен `abs(Δh(x)) ≤ 1/значение`. |
| **Количество сегментов на ребро** | Максимальное количество сегментов на ребре. |
| **Количество сегментов на радиус** | Количество сегментов на радиус. |
| **Второй порядок** | Разрешить сетку второго порядка. |
| **Оптимизировать** | Разрешить оптимизацию. |
| **Разрешить четырёхугольники** | Разрешить [четырёхсторонние грани (англ)](https://en.wikipedia.org/wiki/Types_of_mesh#Two-dimensional). |

### DWG

![](/images/Preferences_Import-Export_Page_DWG.png)

DWG (от англ. drawing - чертеж) - это проприетарный (запатентованный) формат бинарных файлов с закрытым исходным кодом для хранения данных и метаданных 2D и 3D проектирования. Для обработки файлов DWG в FreeCAD требуются внешние программы-конвертеры.

**Примечание:** Все настройки для формата DXF также применимы к DWG.

На этой вкладке вы можете настроить следующие параметры:

| Параметр | Описание |
| --- | --- |
| **Метод преобразования** | Выберите конвертер DWG для использования:  * **Автоматически**: FreeCAD попытается найти конвертер автоматически, следуя порядку, указанному в остальном списке. [представлено в версии 0.21](/Release_notes_0.21/ru "Release notes 0.21/ru"): Для LibreDWG используется путь поиска ОС (Операционной Системы) (`os.getenv(«PATH»)`), для остальных конвертеров предполагаются пути установки по умолчанию. * **LibreDWG**: [LibreDWG](https://www.gnu.org/software/libredwg/) - это библиотека для чтения и записи DWG с открытым исходным кодом. Она не поддерживает несколько структур DWG и не всегда даёт верные результаты. * **ODA Converter**: [ODA Конвертер Файлов](https://www.opendesign.com/guestfiles/oda_file_converter) - бесплатная утилита, предоставляемая Open Design Alliance. Она даёт очень хорошие и надёжные результаты. * **QCAD pro**: [QCAD pro](https://qcad.org/en/qcad-command-line-tools#dwg2dwg) - это платная версия 2D CAD-платформы QCAD DXF с открытым исходным кодом. Его DWG-конвертер использует библиотеки Teigha от OpenDesign Alliance и поэтому даёт такие же хорошие результаты, как и ODA File Converter. |
| **Путь к файлу конвертера** | Если FreeCAD не может найти конвертер, необходимо указать путь:  * LibreDWG: dwg2dxf или dxf2dwg на Linux и macOS, dwg2dxf.exe или dxf2dwg.exe на Windows. Для преобразования как из DWG, так и в DWG можно указать любое имя файла. * ODA Конвертер: ODAFileConverter на Linux и macOS, ODAFileConverter.exe на Windows. * QCAD pro: dwg2dwg (bash скрипт) на Linux и macOS, dwg2dwg.bat на Windows.   [представлено в версии 0.21](/Release_notes_0.21/ru "Release notes 0.21/ru"): Путь должен содержать правильное имя файла. |

### DXF

![](/images/Preferences_Import-Export_Page_DXF.png)

AutoCAD [DXF](/DXF "DXF") (Drawing eXchange Format) - это проприетарный (запатентованный) формат для обмена данными САПР между AutoCAD и другими программами.

На этой вкладке вы можете настроить следующие параметры:

| Параметр | Описание |
| --- | --- |
| **Показывать этот диалок при импорте и экспорте** | Если флажок установлен, этот диалог настроек будет отображаться при импорте или экспорте файлов DXF. |
| **Применить устаревший модуль импорта Python** | Если флажок установлен, будет использоваться импортёр на Python, в противном случае - более новый импортёр на C++. Импортер на C++ работает быстрее, но пока не имеет столько возможностей. Импортёр Python использует **Правка → Настройки... → Набросок(Draft) → Основные → Уровено внутренней точности**. Для получения точного результата импорта установите значение 8 или выше. Для доступа к настройкам, предварительно должен быть запущен верстак Набросок(Draft). |
| **Применить устаревший модуль экспорта Python** | Если флажок установлен, будет использоваться экспортёр на Python, в противном случае - более новый экспортёр на C++. Экспортёр на C++ быстрее, но пока не имеет столько возможностей. |
| **Автоматически загружать и обновлять библиотеки DXF** | Отметив этот флажок, вы разрешите FreeCAD загрузить [Python-конвертер](/FreeCAD_and_DXF_Import/ru "FreeCAD and DXF Import/ru") для импорта и экспорта DXF. Этот конвертер не может поставляться в комплекте с FreeCAD, так как у него другая лицензия на программное обеспечение. |
| **Импорт** | Выберите, что будет импортировано. Если установлен флажок **текст и размеры**, тексты и [mtexts (мультитексты)](https://www.autodesk.com/techpubs/autocad/acad2000/dxf/mtext_dxf_06.htm) также будут импортированы.  Если установлен флажок **точки**, точки также будут импортированы.  Если установлен флажок *Шаблоны (Слои)*, то объекты бумажного пространства также будут импортированы.  Если отмечена опция **\*блоки**, анонимные блоки (имена которых начинаются с символа \*) также будут импортированы. |
| **Создать** | Выберите, что будет создано. Если выбрана опция *простые фигуры деталей*, будут созданы только стандартные объекты деталей. Это самый быстрый вариант.  Если выбрана опция *объекты эскиза*, по возможности будут созданы параметрические объекты эскизов.  Если выбрана опция *Наброски (Эскизы)*, эскизы будут создаваться по мере возможности. |
| **Коэффициент масштабирования для импортируемых файлов** | Scale factor to apply to DXF files on import. The factor is the conversion between the units of your DXF file and millimeters. Example: for files in millimeters: 1, in centimeters: 10, in meters: 1000, in inches: 25.4, in feet: 304.8. |
| **Использовать цвет из файла DXF** | Если флажок установлен, цвета будут по возможности извлекаться из объектов DXF. В противном случае будут применяться цвета по умолчанию. |
| **Присоединить геометрию** | Если флажок установлен, FreeCAD попытается соединить совпадающие объекты в одну кривую. Обратите внимание, что это может занять некоторое время! |
| **Группировать Слои в блоки** | Если флажок установлен, объекты из одних и тех же слоёв будут объединены в Блоки Набросока (Draft Blocks), которые отображаются быстрее, но менее удобны для редактирования. |
| **Использовать стандартный размер шрифта для текстов** | Если флажок установлен, импортированные тексты будут иметь стандартный размер [Набросок Текст (Draft Text)](/Draft_Text/ru "Draft Text/ru"), а не тот, который они имеют в DXF-документе. |
| **Использовать Слои** | Если флажок установлен, слои DXF будут импортироваться как [Слои в Наброске (Draft Layers)](/Draft_Layer/ru "Draft Layer/ru"). |
| **Импорт областей штриховки в виде ломанных (кривых)** | Если флажок установлен, то штриховка будет преобразованна в простые ломанные (кривые). |
| **Рендеринг поли-линий с шириной** | Если флажок установлен, то если у полилиний определена ширина (толщина линии), они будут отображаться как замкнутые кривые с правильной шириной. |
| **Считать эллипсы и сплаайны полилиниями** | Экспорт эллипсов и сплайнов плохо поддерживается. Используйте эту опцию, чтобы экспортировать их как полилинии. Параметр **Макс сплайн сегментов** - это максимальная длина каждого из сегментов полилинии. Если установить значение **0**, то весь сплайн будет рассматриваться как прямой сегмент. |
| **Экспорт 3D-объектов в виде многогранных сеток** | Если флажок установлен, все объекты, содержащие грани, будут экспортированы как 3D многогранники. |
| **Экспортировать чертёжные виды как блоки** | Если этот флажок установлен, TechDraw Views (Виды Чертежа) будут экспортироваться в виде блоков. Это может не сработать для шаблонов Post DXF R12. |
| **Проецировать экспортированные объекты вдоль текущего направления просмотра** | Если флажок установлен, экспортируемые объекты будут проецироваться с учетом текущего направления просмотра. Эта опция работает только при использовании устаревшего экспортёра Python. |

### IFC

[Industry Foundation Classes (англ.)](http://en.wikipedia.org/wiki/Industry_Foundation_Classes) [(рус.)](https://ru.wikipedia.org/wiki/Industry_Foundation_Classes) (IFC) - это широко распространённый формат для обмена данными между [BIM (англ.)](http://en.wikipedia.org/wiki/Building_Information_Modeling) [(рус.)](https://ru.wikipedia.org/wiki/BIM) приложениями. Он используется в архитектуре и инженерном деле.

Примечание для пользователей Linux: Для работы с этим форматом файлов FreeCAD требуется модуль [IfcOpenShell (Открытая оболочка IFC)](/Extra_python_modules/ru "Extra python modules/ru").

#### Импорт IFC

![](/images/Preferences_Import-Export_Page_IFC_import.png)

На этой вкладке вы можете настроить следующие параметры:

| Параметр | Описание |
| --- | --- |
| **Показывать этот диалог при импорте** | Если флажок установлен, то при импорте файлов IFC будет показан этот диалог настроек. |
| **Показывать отладочные сообщения** | Показывает подробные отладочные сообщения при импорте и экспорте файлов IFC в [Панели "Отчёт"](/Report_view/ru "Report view/ru"). |
| **Создавать клоны, если объекты имеют общую геометрию** | Объекты IFC могут разделять одно и то же геометрическое пространство между несколькими объектами, различаясь только их размещением. Когда эта опция включена, для достижения того же результата в FreeCAD используются клоны. Один объект становится базовым, остальные - его клонами. |
| **Количество используемых ядер (экспериментальная)** | Укажите количество ядер процессора, которое будет использоваться для импорта IFC. Максимальное число должно быть меньше, чем количество реально доступных ядер. Используйте **0**, чтобы отключить эту функцию. |
| **Импорт архитектурных IFC-объектов как** | Что будет создано в FreeCAD для архитектурных объектов IFC. |
| **Импорт структурных IFC-объектов как** | Что будет создано в FreeCAD для структурных объектов IFC. |
| **Корневой элемент** | Будут импортированы только подтипы указанного элемента. Сохраните предопределённый элемент [IfcProduct](http://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifckernel/lexical/ifcproduct.htm) для импорта всех элементов здания. |
| **Разъединить отверстия** | Если флажок установлен, проёмы будут импортированы как вычитания, в противном случае стены будут иметь уже вычитанные проёмы. |
| **Обнаружить выдавливания** | Если флажок установлен, программа импорта будет пытаться обнаружить выдавливание. Обратите внимание, что это может замедлить работу. |
| **Разделить многослойные стены** | Разделяет стены, состоящие из нескольких слоев. |
| **Добавлять префикс идентификатора ID к именам** | Если флажок установлен, имена объектов будут снабжены префиксом из номера [IFC ID](http://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcutilityresource/lexical/ifcgloballyuniqueid.htm). |
| **Объединить материалы с одинаковым именем и цветом** | Если в файле IFC найдено несколько материалов с одинаковым именем, они будут рассматриваться как один. |
| **Импорт свойств IFC в электронную таблицу** | Если флажок установлен, каждый объект будет иметь свои [свойства IFC](http://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcpropertyresource/lexical/ifcproperty.htm), сохраняемые в электронной таблице. |
| **Разрешить недопустимые фигуры** | Если флажок снят, недопустимые фигуры не импортируются. |
| **Список исключений** | Список [IFC-объектов](https://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/toc-5.htm), разделённый запятыми, который необходимо исключить из импорта. |
| **Вписать в вид при импорте** | Подогнать вид во время импорта на импортируемые объекты. Это замедлит импорт, но можно наблюдать за импортом. |
| **Импортировать полные параметрические определения FreeCAD, если они доступны** | Создаёт полную параметрическую модель при импорте, используя сохранённые свойства объектов FreeCAD. Чтобы получить свойства FreeCAD, модель должна быть экспортирована с использованием опции **Экспорт полной параметрической модели FreeCAD**. |
| **Заменить 'Проект', 'Местность', 'Здание' и 'Этаж' на 'Группу'** | Если флажок установлен, группы будут использоваться для замены упомянутых объектов. |

#### Экспорт IFC

![](/images/Preferences_Import-Export_Page_IFC_export.png)

На этой вкладке вы можете настроить следующие параметры:

| Параметр | Описание |
| --- | --- |
| **Показывать этот диалог при экспорте** | Если флажок установлен, то при экспорте файлов IFC будет показан этот диалог настроек. |
| **Тип IFC экспорта** | Выберите способ экспорта модели: как **Стандартная модель**, **Структурный анализ** или **Стандартная + Структурная**. |
| **Принудительно экспортировать как Brep** | Некоторым программам просмотра IFC не нравятся объекты, экспортированные в виде выдавливания (экструзии). Используйте этот параметр, чтобы заставить все объекты экспортироваться как геометрия [Граничное представление (рус.)](https://ru.wikipedia.org/wiki/Граничное_представление) [BREP (англ.)](https://en.wikipedia.org/wiki/Boundary_representation). Но по возможности избегайте экспорта в формате BREP, так как это делает объекты непараметрическими. |
| **Использовать параметры триангуляции DAE** | Использовать параметры триангуляции, установленные на странице параметров DAE. |
| **Объединять компланарные грани при триангуляции** | Изогнутые формы, которые не могут быть представлены как кривые в IFC, декомпозируются на плоские грани. Если этот параметр отмечен, то для соединения компланарных (совмещённых) граней выполняются дополнительные вычисления. |
| **Хранить уникальный ID для IFC в объектах FreeCAD** | При экспорте объектов без [уникального идентификатора ID (англ.)](http://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcutilityresource/lexical/ifcgloballyuniqueid.htm) (UID), сгенерированный UID будет сохранён внутри объекта FreeCAD для повторного использования при следующем экспорте объекта. Это приводит к уменьшению различий между версиями файлов. |
| **Использовать сериализатор IfcOpenShell, если доступен** | [IFCOpenShell](/Extra_python_modules/ru#IfcOpenShell "Extra python modules/ru") - это библиотека для файлов IFC. Её функция *сериализатора* может создавать корректную геометрию IFC из фигур [OCC](/Glossary/ru#OCC "Glossary/ru"). Обратите внимание, что это всё ещё экспериментальная функция. |
| **Экспорт двумерныъх объектов как IfcAnnotations** | Если флажок установлен, 2D-объекты будут экспортироваться как [IfcAnnotation (англ.)](https://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcproductextension/lexical/ifcannotation.htm). |
| **Экспорт полной параметричекой модели FreeCAD** | Если флажок установлен, все свойства объектов FreeCAD будут сохранены внутри экспортированных объектов, что позволит воссоздать полную параметрическую модель при повторном импорте с помощью опции **Импортировать полные параметрические определения FreeCAD, если они доступны**.. |
| **Повторное использование аналогичных записей** | Если флажок установлен, похожие элементы будут использоваться в файле только один раз, если это возможно. Это может значительно уменьшить размер файла, но сделает его менее удобочитаемым. |
| **Отключить IfcRectangleProfileDef** | По возможности объекты IFC, представляющие собой выдавленные прямоугольники, будут экспортироваться как [IfcRectangleProfileDef (англ.)](http://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcprofileresource/lexical/ifcrectangleprofiledef.htm). Для приложений, у которых возникают проблемы с импортом этих элементов, выберите этот параметр, чтобы все профили экспортировались как [IfcArbitraryClosedProfileDef (англ.)](http://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcprofileresource/lexical/ifcarbitraryclosedprofiledef.htm). |
| **Автоопределение и экспорт в качестве стандартных случаев, когда это применимо** | Некоторые типы IFC, такие как [IfcWall](https://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcsharedbldgelements/lexical/ifcwall.htm) или [IfcBeam](https://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcsharedbldgelements/lexical/ifcbeam.htm) имеют специальные стандартные версии, такие как [IfcWallStandardCase](https://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcsharedbldgelements/lexical/ifcwallstandardcase.htm) или [IfcBeamStandardCase](https://standards.buildingsmart.org/IFC/RELEASE/IFC4/ADD1/HTML/schema/ifcsharedbldgelements/lexical/ifcbeamstandardcase.htm). Если эта опция выбрана, FreeCAD будет автоматически экспортировать такие объекты как стандартные случаи при выполнении необходимых условий. |
| **Добавить местность по умолчанию, если она не найдена в документе** | При экспорте файла IFC, если в документе FreeCAD не найдено ни одной площадки, она добавляется по умолчанию. В соответствии со стандартом IFC наличие площадки не является обязательным, но обычно принято иметь в файле хотя бы одну локацию. |
| **Добавьте этаж здания по умолчанию, если он не найден в документе** | При экспорте IFC-файла, если в документе FreeCAD не найдено ни одного этажа здания, он добавляется по умолчанию. Этажность здания не является обязательной в соответствии со стандартом IFC, но общепринятой практикой является наличие хотя бы одного этажа в файле. |
| **Единицы файлов IFC** | Выберите, какие единицы измерения будут использоваться при экспорте файлов IFC. |
| **Добавить здание по умолчанию, если оно не найдено в документе** | При экспорте файла IFC, если в документе FreeCAD не найдено ни одного здания, оно добавляется по умолчанию. **Внимание**: Стандарт IFC требует наличия хотя бы одного здания в каждом файле. Отключение этой опции приведет к созданию нестандартного файла IFC.  Однако мы в FreeCAD считаем, что наличие здания не должно быть обязательным, и эта опция создана для того, чтобы иметь возможность продемонстрировать нашу точку зрения. |
| **Экспортировать вложенные группы как сборки** | В FreeCAD есть возможность вложить группы внутрь зданий или этажей. Если эта опция отключена, то группы FreeCAD будут сохраняться как IfcGroups и объединяться в структуру здания. Однако агрегирование элементов, не относящихся к зданию, таких как IfcGroups, не рекомендуется стандартами IFC. Поэтому можно экспортировать эти группы как IfcElementAssemblies, что позволит получить файл, совместимый с IFC. Однако мы в FreeCAD считаем, что вложение групп внутри структур должно быть возможным, и эта опция создана для того, чтобы продемонстрировать нашу точку зрения. |

| **Какой объект импортировать** | Выберите, какие объекты должны быть импортированы и как. Если выбрана опция **Объект результата VTK**, будет импортирован объект FreeCAD МКЭ (FEM) VTK (аналогичный экспортированному объекту).  Если выбрана опция **Объект сетки МКЭ**, результаты в VTK-файле будут опущены, будут импортированы только данные сетки, и будет создан объект сетки МКЭ FreeCAD.  Если выбрано значение **Объект результата FreeCAD**, импортированные данные будут преобразованы в объект результата FreeCAD МКЭ (FEM). **Примечание:** эта настройка требует точных имён компонентов результата, поэтому она работает правильно только с VTK-файлами, экспортированными из FreeCAD. |
